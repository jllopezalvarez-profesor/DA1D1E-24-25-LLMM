Ejemplos de pseudoclases

De interacción

Al pasar por encima (:hover), al hacer clic (:active) y al hacer foco (:focus)

a.a-interaccion:hover {
            color: lightgreen;
            background-color: black;
            }
            a.a-interaccion:active {
            color: chartreuse;
            background-color: coral;
            }
            a.a-interaccion:focus {
            color: crimson;
            background-color: lightskyblue;
            }

Enlace que no va a ningún sitio

Al hacer foco en elemento interior (:focus-within)

p.p-focus-within {
            background-color: lightgreen;
            padding: 30px;
            }
            p.p-focus-within:focus-within {
            background-color: mediumseagreen;
            }

Enlace que no va a ningún sitio

En este caso, la pseudoclase está en el párrafo que contiene el enlace. Cuando se hace foco en el enlace, su contenedor (el p), cambia.

De estructura

Primer hijo (:first-child) y último hijo (:last-child) de un elemento.

ul.first-last-child :first-child {
            background-color: mediumseagreen;
            color: green;
            }
            ul.first-last-child :last-child {
            background-color: mediumblue;
            color: lightblue;
            }

Aunque aquí sólo hay li dentro del padre (ul), si tuviéramos un contenedor con otros tipos de elementos mezclados, se elige el primero o el último, independientemente del tipo de elemento.

Por eso toda la lista interior está en color azul, porque es el último del segundo li de la lista exterior.

Esto pasa porque :first-child equivale a *:first-child. Si se usar li:first-child, no se comporta de la misma manera

Si se quiere hacer con tipos específicos, usar first-of-type o last-of-type, o usar elemento:first-child o elemento:last:child

De formularios - estado

:disabled, :enabled, :read-only, :read-write

input:read-only,
            select:read-only {
            background-color: crimson;
            color: white;
            }
            input:disabled,
            select:disabled {
            background-color: gray;
            color: white;
            }

Los select no pueden tener el atributo "readonly" (no lo permite el estándar).Son por naturaleza "read-only". Son en realidad inmutables, no podemos cambiar sus valores, solo seleccionar uno. Aparecen con estilos read-only (si se ha creado una regla), aunque no tenga atributo "readonly". En los select disabled no se puede interactuar con la lista, pero en los readonly sí.

Ojo con disabled y read only. El orden de las pseudoclases en el css importa.

Disabled se considera, siempre, también "read-only". Tiene sentido, porque un campo deshabilitado es como un read only, solo que limita aún más la interacción.

Si ponemos los estilos para read-only después de los de disabled, veremos que los de disabled no aplican. Porque como todo disabled es read-only, se aplica la última regla.

Hay que usar el orden adecuado, o selectores más específicos

De formularios - validación

:required, :user-invalid

En general, es mejor usar ":user-invalid" frente a ":invalid". Si se usa ":invalid" el campo aparece como erróneo desde el primer momento. Con ":user-invalid" aparece como erróneo sólo si el usuario ha interactuado con el campo.
